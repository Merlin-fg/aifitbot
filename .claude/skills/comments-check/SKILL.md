# Comments Check Skill

三个维度审查代码注释质量，输出分级报告。

## 触发

```
/comments-check                     → 审查上一次改动的文件
/comments-check src/services/auth_service.py → 审查指定文件
/comments-check src/routes           → 审查指定目录
/comments-check --all                → 全项目审查
```

## 审查维度

1. **覆盖率** — 每个函数/重要代码块是否有注释（docstring + 行内注释），目标 30% 注释率
2. **准确性** — 注释描述是否与代码实际逻辑一致，有无过时/错误注释
3. **小白可读性** — 注释是否用通俗语言解释，避免专业术语堆砌，新手能否看懂

## 执行流程

1. 读取目标文件
2. 统计每个文件的注释行/总行比
3. 逐函数审查注释三个维度
4. 输出分级报告

## 输出格式

```
══════════════════════════════════════
  AIFitBot 注释质量审查报告
══════════════════════════════════════
文件: src/services/auth_service.py
行数: 120    注释行: 45    注释率: 37.5%

🔴 严重 (1)
  L26: 注释"将明文密码哈希化"未提及 SHA-256 预哈希逻辑
  → 建议: "将明文密码哈希化（超72字节时先SHA-256预哈希避免截断）"

🟡 警告 (2)
  L71: change_password 缺少返回值说明
  L89: _create_token 缺少参数文档

🟢 建议 (1)
  L10: "bcrypt 密码加密上下文" → 建议改为 "bcrypt 加密工具，用于密码的加盐哈希"
──────────────────────────────────────
合计: 1严重  2警告  1建议
══════════════════════════════════════
```

## 注释率标准

| 注释率 | 评级 |
|--------|------|
| > 25% | ✅ 良好 |
| 15-25%| ⚠️ 偏低 |
| < 15% | ❌ 不足 |
